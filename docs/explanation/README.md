# Explanation

Use this section for conceptual and architectural context.

An explanation page should:

- Clarify why the system works the way it does.
- Cover tradeoffs, alternatives, and constraints.
- Help readers build accurate mental models.
- Connect design choices to user and runtime outcomes.

Typical examples:

- Execution model and preview/commit rationale
- Core/compiler separation and protocol-agnostic design
- State persistence architecture decisions

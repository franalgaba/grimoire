# How-to Guides

Use this section for task-oriented procedures.

A how-to guide should:

- Start with the outcome the reader wants.
- Provide exact steps to reach that outcome.
- Include prerequisites and expected results.
- Avoid deep theory unless required for the task.

Typical examples:

- "Run the test suite"
- "Bridge assets with Across"
- "Publish a new release"
